# SaaS Revenue Intelligence

An MCP server that estimates the Annual Recurring Revenue (ARR) of any SaaS company from its domain. Deployed on [Context Protocol](https://ctxprotocol.com) as a paid tool — query it with a domain, get a calibrated ARR estimate with confidence score and signal breakdown.

<!-- **Mean error: 1.35x** across 60 public SaaS companies with known ARRs. -->

---

## How It Works

Five signals are fetched concurrently and fed into a Ridge regression model trained on S-1 and public ARR data:

| Signal | Source | What it measures |
|--------|--------|-----------------|
| Headcount | Crustdata → PDL → SerpAPI → LinkedIn | Revenue capacity (primary signal) |
| Hiring velocity | Ashby → Greenhouse → Lever → Workable → BambooHR → Rippling | Growth rate |
| Web traffic | Tranco top-1M rank | Brand reach |
| Review momentum | G2 / Trustpilot (+ Crustdata fallback) | Customer acquisition |
| Pricing tier | Scraped pricing page | ACV estimate |

Results are cached in Redis for 24 hours. Cold fetches take 10–25s. Cached responses return in under 3s.

## Setup

**Requirements:** Python 3.10+, Redis (via Upstash)

```bash
git clone https://github.com/your-username/saas-revenue-intelligence
cd saas-revenue-intelligence
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Mac/Linux
pip install -r requirements.txt
```

Copy `.env.example` to `.env` and fill in your keys:

```bash
cp .env.example .env
```

Train the model (required before first run):

```bash
python -m model.weights
```

Start the server:

```bash
python server.py
```

## Model

Ridge regression (RidgeCV with leave-one-out cross-validation) fit on log-transformed features:

| Feature | Description |
|---------|-------------|
| `log(headcount)` | Scale in log space — primary signal |
| `log(open_roles)` | Growth velocity |
| `rank_score` | Web reach (Tranco) |
| `momentum` | Review momentum |
| `acv_norm` | Pricing tier |
| `headcount × rank` | Scale × reach interaction |

Training excludes companies with ARR > $5B. Retrain after adding rows to `training_data.json`:

```bash
python -m model.weights
```

## Signal Fallback Chain

**Headcount:** Crustdata → PDL → LinkdAPI → Proxycurl → SerpAPI → ScraperAPI → LinkedIn direct → `0`

**Hiring:** Ashby → Greenhouse → Lever → Workable → BambooHR → Rippling → `0`

**Reviews:** G2 → Trustpilot → Crustdata G2 fallback → `empty`

**Traffic:** Tranco list downloaded once, disk-cached for 3 days in `.tranco/`

**Pricing:** Scraped pricing page → heuristic extraction → `unknown`
